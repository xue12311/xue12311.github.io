华为推送 - 角标功能

一开始打算通过HmsMessageService监听收到的推送消息，来修改角标内容，但是onMessageReceived() 方法只支持通透消息的监听,不能符合我们的业务需求，后面通过仔细研究开发文档，发现角标可以通过服务端设置。

参数值 取值类型 必选(M)/ 可选(O) 功能
add_num integer O 角标累加数字，大于0小于100的正整数。
class string O 应用包名+应用入口Activity类名。样例：com.huawei.codelabpush.MainActivity
set_num integer O 角标设置数字，大于等于0小于100的整数。如果set_num与add_num同时存在时，以set_num为准。

APN、VPN 专网切换

  1. 通过shell命令ping项目专网是否可以正常访问，若能ping通即当前处于内网状态，若不能ping通即当前处于外网状态。
  2. APN 网络切换
    2.1 通过与移动公司合作，为客户提供专门的SIM卡。
    2.2 运营人员会去检察院教导客户设置对应APN专网。
    2.3 Android会在用户点击首页时判断用户是否已经切换到专网，若处于外网状态，应用会自动跳转到手机APN设置界面，引导用户设置切换APN专网。
  3. VPN网络切换
    3.1 通过与深信服合作，使用他们提供的SDK，实现VPN网络切换功能
    3.2 用户在第一次切换内网时，弹出VPN登录提示框，用户需要输入对应的VPN账号进行VPN登录操作，登录成功后，自动切换到专网环境，保存VPN账号到本地( )，跳转到登录界面。
    3.2 用户在第一次切换内网时，弹出VPN登录提示框(若本地已有保存的VPN账号，自动切换到专网)，用户需要输入对应的VPN账号进行VPN登录操作，登录成功后，自动切换到专网环境，保存VPN账号到本地( SharedPreferences )。

进程保活

  1. 在低版本android手机上通过广播监听（解锁屏广播，网络状态广播）来启动后台服务。双进程互相拉起。
  2. 提高Service的优先级 （将onStartCommand 设置为 START_STICKY 确保在系统内存足够时，能够做到Service自动重启），设置Service为前台服务。
  3. 引导用户关闭省点模式2，加入系统白名单，允许应用后台运行。
  4. 启用多媒体锁 (提高播放一段无声音乐) 达到保活效果。

内存泄漏

  1. 在使用Context时注意，减少使用 Activity中的Context，尽量选择使用 applicationContext。
  2. 在不使用静态变量时将其设置为null。
  3. 在单例模式下，减少使用Activity作为参数。
  4. 在使用Handler时，可以通过 静态内部类+弱引用（WeakHandler）的方式使用，在Activity被销毁时使用removeCallbacksAndMessages()方法清空消息。
  5. 及时关闭资源对象，关闭输入输出流，及时回收Bitmap，及时销毁WebView，及时注销监听器。
  6. 在使用MVP设计模式时,由于Presenter经常需要执行耗时操作，当耗时操作未完成时就关闭Activity，可能会导致Presenter一直持有Activity的引用，从而造成内存泄漏。所以在开发时我们会在BasePresenter中采用弱引用（WeakReference）的方式持有View，在Activity被销毁时及时解除引用。
  7. 在使用协程(coroutine)进行网络请求时，请及时取消协程。也可以考虑通过ViewModelScope，LifecycleScope等方式

FutureBuilder ---- 网络请求，数据库读取 更新 界面

  • FutureBuildere是一个将异步操作和异步UI更新结合在一起的类，通过它我们可以将网络请求，数据库读取等的结果更新到页面上
  • FutureBuilder的构造方法
  • future ：Future对象表示此构造器当前连接的异步计算
  • initialData ：表示一个非空的Future完成前的初始化数据
  • builder ：AsyncWidgetBuilder类型返回的函数，是一个基于异步交互构建widget的函数
    builder:
    BuildContext context
    AsyncSnapshot snapshot
    connectionState 表示与异步计算的连接状态
    data - 异步计算接收的最新数据
    error - 异步计算接收的最新错误对象；
ConnectionState ：与异步计算的连接状态
  • none ：future还未执行的快照状态
  • waiting ：连接到一个异步操作，并且等待交互，一般在这种状态的时候，我们可以显示加载框
  • active ：连接到一个活跃的操作，比如stream流，会不断地返回值，并还没有结束，一般也是可以显示加载框
  • done：异步操作执行结束，一般在这里可以去拿取异步操作执行的结果，并显示相应的布局
ConnectionState 当前没有连接到任何的异步任务
ConnectionState.none 当前没有连接到任何的异步任务
ConnectionState.waiting 连接到异步任务并等待进行交互
ConnectionState.active 连接到异步任务并开始交互
ConnectionState.done 异步任务中止
